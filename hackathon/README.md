**A 2.5-week AI developer hackathon running May 13 – May 30, 2026, kicked off live during [Agent Academy Live](https://aka.ms/agent-academy) on May 12.**

Apply what you've learned from the [Agent Academy](https://github.com/microsoft/agent-academy) curriculum to build real, working AI agents using [Microsoft Copilot Studio](https://www.microsoft.com/en-us/microsoft-copilot/microsoft-copilot-studio). ⚠️ **Important**: Before submitting, read our [Disclaimer](./DISCLAIMER.md) — do not upload confidential information.

> 📚 Looking for the Agent Academy curriculum? Visit the [main README](../README.md).

## What's the Agent Academy Live Hackathon?

A connected program that validates the Agent Academy curriculum by converting learning into real, working AI agents:

1. **📚 Learn** — Engage with [Agent Academy](https://aka.ms/agent-academy) curriculum modules
2. **📺 Live** — Agent Academy Live delivers structured instruction, demos, and inspiration
3. **🔨 Build** — Apply what you learned to build real agents
4. **📤 Share** — Submit demos and architectures for review
5. **🔄 Improve** — Your feedback and insights inform curriculum and product evolution

**Open to everyone** — all skill levels welcome! Choose a participation track that matches your experience.

## Participation Tracks

Tracks are self-selected starting levels (like belt or weight classes). They set expectations for judging — but awards are issued across the full field, not separately per track.

| Track | Level | What "Excellent" Looks Like |
|-------|-------|---------------------------|
| 🟢 **Recruit** | Beginner | First working agent demonstrating one or more Agent Academy concepts. Simple scope done well, clear learning, a working end-to-end demo. |
| 🟡 **Operative** | Intermediate | Solving a real-world problem using multiple Agent Academy patterns. Multi-pattern solution, reasonable robustness, thoughtful tradeoffs. |
| 🔴 **Commander** | Advanced | Advanced or enterprise-oriented design including integrations, security, or governance considerations. Production-minded decisions with clear enterprise constraints. |

## Award Categories & Prizes

Total prize pool: **$12,000 in Microsoft Store credit**. All awards are judged by a panel — no community voting.

| Award | Prize | What Judges Look For |
|-------|-------|---------------------|
| 🏅 **Best Use of Agent Academy Patterns** | $2,500 | Correct, intentional, well-explained use of Agent Academy concepts and patterns |
| 🌍 **Best Real-World Use Case** | $2,500 | Solving a meaningful problem with a believable path to adoption |
| ⚙️ **Best Technical Execution** | $2,000 | Build quality — reliability, correctness, and engineering discipline |
| 🏢 **Best Enterprise-Ready Design** | $2,000 | Designs ready to operate in real organizational environments |
| 🎨 **Most Creative Solution** | $1,500 | Novelty in approach or experience while delivering a coherent, working agent |
| ⭐ **Judges' Choice** | — | Exceptional submission that stands out for overall excellence |

## How to Participate

### 1. Prepare (Before May 12)

Complete core [Agent Academy](https://aka.ms/agent-academy) modules (recommended, not required):

- **Recruit** courses — foundational agent concepts
- **Operative** courses — multi-pattern agent design
- **Commander** courses — enterprise agent architecture

### 2. Attend the Kickoff — May 12

Join **Agent Academy Live** for the official hackathon kickoff:

- Challenge walkthrough and rules overview
- Submission requirements and examples
- Office hours and AMA schedule announcement

### 3. Build Your Agent — May 13–30

- Choose your track (Recruit, Operative, or Commander)
- Build a real agent applying Agent Academy patterns
- Create a 5-minute or less demo video
- Get help at Office Hours and AMAs during the build window

### 4. Submit Your Project

**Submission Deadline**: Friday, May 30, 2026 (11:59 PM PT)

Open a new issue in this repo using the [🏆 Project Submission template](../../issues/new?template=project.yml)

⚠️ **Before submitting**:
- Read the [Disclaimer](./DISCLAIMER.md) — No confidential information allowed
- Review the [Code of Conduct](../CODE_OF_CONDUCT.md)
- Ensure your repository is public and includes a README

## Submission Requirements

Each submission must include:

- **Project title**
- **Selected track** (Recruit, Operative, Commander)
- **Problem statement** — What problem does your agent solve?
- **Agent Academy modules or concepts used** — Which patterns did you apply?
- **Architecture overview** — Diagram or written description
- **Demo video** — 5 minutes or less showing your solution
- **Curriculum feedback** — What worked well? What was unclear or difficult?

## Judging

All submissions are scored using category-specific rubrics (80%) plus a Track Adjustment (20%) that accounts for the participant's selected starting level.

<details>
<summary><strong>📋 Track Adjustment (20% — applies to all awards)</strong></summary>

| Score | Description |
|-------|-------------|
| 5 | Far exceeds expectations for the selected track |
| 4 | Exceeds expectations |
| 3 | Meets expectations |
| 2 | Partially meets expectations |
| 1 | Does not meet expectations |

**By track:**
- **Recruit**: Simple scope done well; clear learning; a working end-to-end demo
- **Operative**: Multi-pattern solution to a real problem; reasonable robustness; thoughtful tradeoffs
- **Commander**: Production-minded decisions (integration, security/privacy, governance, monitoring/eval); clear enterprise constraints

</details>

<details>
<summary><strong>🏅 Best Use of Agent Academy Patterns (80% category criteria)</strong></summary>

| Criteria | Weight | Scoring (1–5) |
|----------|--------|---------------|
